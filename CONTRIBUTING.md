# Contributing to PMM CI Hackathon

Thank you for your interest in contributing! This document provides guidelines and instructions to help you get started.

## How to Contribute

### 1. Find or Create an Issue

- Browse the [open issues](../../issues) to find something to work on
- If you have a new idea, open an issue to discuss it before starting work
- Comment on an issue to let others know you're working on it

### 2. Fork and Clone

```bash
git clone https://github.com/<your-username>/pmm-ci-hackathon.git
cd pmm-ci-hackathon
```

### 3. Create a Branch

Use a descriptive branch name:

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/issue-description
```

### 4. Make Your Changes

- Keep changes focused and minimal
- Add documentation for any new tools or scripts
- Include sample data or usage examples where helpful

### 5. Submit a Pull Request

- Push your branch and open a Pull Request against `main`
- Describe what your PR does and link any related issues
- Request a review from a maintainer

## Project Areas

Contributions are welcome in any of these areas:

| Area | Description |
|------|-------------|
| `src/` | Scripts and tools for data collection and analysis |
| `data/` | Sample datasets, schemas, and data dictionaries |
| `docs/` | Guides, tutorials, and reference documentation |

## Code Style

- Use clear, descriptive variable and function names
- Include comments for complex logic
- Keep functions small and focused on a single responsibility

## Questions?

Open an issue or start a discussion — we're happy to help!
