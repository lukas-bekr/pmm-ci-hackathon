# Competitive Map

| Category | Competitors | Notes |
|----------|-------------|-------|
| **Enterprise data services** | [Zyte](https://zyte.com), [Import.io](https://import.io), [Mozenda](https://mozenda.com) (merged with Dexi.io) | High-priced turn-key "data-as-a-service" for large companies. Compete with Apify-managed. Lower margin, harder to scale than SaaS/PaaS. |
| **Vertical scraping solutions** | [PhantomBuster](https://phantombuster.com), [Import.io](https://import.io) | Started horizontal, gravitated to specific use cases (lead gen, e-commerce). Sacrificed flexibility and DX. |
| **Point-and-click tools** | [Dexi.io](https://dexi.io), [Dashblock](https://dashblock.com), [Axiom.ai](https://axiom.ai) | Handy for simple use cases, hard to scale. Appeal to non-technical users for small/one-time projects. High churn, companies often fail. |
| **AI data extractors** | [DiffBot](https://diffbot.com), [Reworkd](https://reworkd.ai) | Use AI to extract structured data automatically. Works for standardized pages (articles, products), not flexible enough for diverse use cases. |
| **API & agent marketplaces** | [RapidAPI](https://rapidapi.com), [agent.ai](https://agent.ai), [Smithery](https://smithery.ai) | Aggregate third-party APIs/agents but limited control over execution, data flows, custom logic. Discovery channels, not platforms. |
| **Scraping APIs** | [Firecrawl](https://firecrawl.dev), [ScrapingBee](https://scrapingbee.com), [ScraperAPI](https://scraperapi.com), [Zyte API](https://zyte.com) | APIs to fetch HTML with anti-blocking. Low-level, no custom compute/storage/processing - customers must host own code. |
| **Proxy vendors** | [Bright Data](https://brightdata.com), [Oxylabs](https://oxylabs.io), [NetNut](https://netnut.io) | Sell residential/datacenter IPs - now a commodity with price competition. Some add extractors/datasets but limited and inflexible. |
| **Web indexes** | [Exa](https://exa.ai), [Tavily](https://tavily.com), [DiffBot](https://diffbot.com), [Perplexity Sonar](https://perplexity.ai), [Bright Data](https://brightdata.com), [Parallel.ai](https://parallel.ai) | Building web indexes for AI context. Won't achieve true knowledge-graph of all information - room for specialized Actors. |
| **Sandbox & browser infra** | [BrowserBase](https://browserbase.com), [Browserless](https://browserless.io), [E2B](https://e2b.dev), [Daytona](https://daytona.io), [Modal](https://modal.com) | Container infrastructure for AI-generated code and browsers. Low-level utilities without marketplace/distribution. |
| **Developer platforms** | [Replit](https://replit.com), [GitHub](https://github.com), [Vercel](https://vercel.com) | Host/version code with execution environments. General-purpose, no standard form factor, not optimized for automation workloads. |
| **Open-source tools** | [Puppeteer](https://pptr.dev), [Playwright](https://playwright.dev), [Selenium](https://selenium.dev), [BeautifulSoup](https://beautiful-soup-4.readthedocs.io), [Scrapy](https://scrapy.org) | Useful libraries, not full solutions. Apify supports them all. |
